## PERMANENT RULES (carry forward)

1. **Shelf detection**: use regex `\bel\b` — never use `'el'` in string (matches "shelf")
2. **Avoid status**: rely ONLY on stored `t.al` field. Never use `NA[t.c]===0` as fallback
3. **Date parsing**: direct part extraction (YYYY-MM-DD split). Never use `new Date("YYYY-MM-DD")` — UTC midnight shift bug in UTC+2
4. **Allocation cutoff**: Apr 13 2026. Pre-cutoff = V1; post-cutoff = V2. Avoid brackets in V2 = 0%, not 1%
5. **CSV schema (Omega)**: Subset, Setup, Ticker, Date, Strike, StopHit, PriceCat, Entry, Stop, Exit, R, WL_gross, RNet, WL_net
6. **GrossR** = (Entry − Exit) / (Stop − Entry); **NetR** = Gross × 0.80 (win) / × 1.20 (loss); **AccR** = alloc × NetR
7. **PowerShell**: Use semicolons not `&&` to chain commands. Use `Out-File -Encoding utf8` not `>`
8. **Template literals**: Use backticks when HTML contains single-quotes
9. **Firebase sanitization**: Destructure-omit rather than set to undefined
10. **Omega Forecast tab preservation**: Any Omega rebuild must preserve the Forecast `<script type="module">` + CSS + HTML view + tab button
11. **Fee Overview box**: Always compute fee % dynamically from `FD.fee_records` + `FD.trade_records` — never read pre-baked `FD.meta.median_trade_fee_pct`
12. **buildW10 field names**: Returns `{date, daily_ar, worst_ticker, worst_tk_ar, pct_of_day, n_trades}` — no `tickers` array field

## BUGS ENCOUNTERED AND RESOLVED (2026-04-24)

### 1. Omega Forecast tab — `window.DAY` scoping bug
- **Symptom**: "No historical data loaded" alert on Forecast tab
- **Root cause**: `let DAY` is block-scoped inside a `<script>` tag, `window.DAY` in the Forecast module was always undefined (3 locations: lines ~4193, 4035, 4047)
- **Fix**: Changed `window.DAY` → `DAY` in all 3 references

### 2. Omega Forecast tab — UTF-8 encoding mojibake
- **Symptom**: Characters like `ÔûÂ`, `ÔÜá`, `ÔÇö` appearing in Forecast tab
- **Root cause**: UTF-8 double-encoded through Latin-1/Windows-1252
- **Fix**: Replaced all mojibake sequences: `ÔûÂ→▸`, `ÔÜá→⚠`, `ÔÇö→—`, `ÔÇô→–`, `┬À→·`, `Ô£ô→✓`

### 3. Portfolio v6 — `renderWorst10` TypeError
- **Symptom**: `Cannot read properties of undefined (reading 'forEach')`
- **Root cause**: `buildW10()` returns `worst_ticker` (string), not `tickers` (array). Render function tried `a.tickers.forEach(...)`
- **Fix**: Changed to direct string access: `if(a&&a.worst_ticker)tickers.push(a.worst_ticker)`
